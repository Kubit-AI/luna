## 2. Minimum-change tier

`bootstrap-file`

Logfire is OTel-native and installs its own `TracerProvider` inside
`logfire.configure()` via `trace.set_tracer_provider(...)`. Python's
OTel API enforces a set-once latch on the global provider, so if
kubit-otel registers first, Logfire's registration silently becomes a
no-op and Logfire spans are lost. The wiring in §3 therefore must run
**after** `logfire.configure()` — either via the merge form
(`additional_span_processors=[...]`, vendor-documented, preferred) or
via the standalone form, which calls `attach()` and fails loudly if
invoked before Logfire initializes.

## 3. Bootstrap snippet

Reference Kubit wiring — the *minimum* code that must end up in the
program (import + processor/`configure` call, plus any assertions
below).

Logfire's canonical extension API is the `additional_span_processors=`
kwarg on [`logfire.configure()`](https://logfire.pydantic.dev/docs/reference/api/logfire/).
Use the **merge form** whenever a `logfire.configure(...)` call is
found per §3a — it's one call-site edit and avoids a second file.
The `processors=` kwarg is deprecated and is **not** a drop-in for
`additional_span_processors=`; if the user is pinned to an older
Logfire, leave `processors=` alone and use the standalone form
instead. Use the **standalone form** when §3a finds no wiring site.

### Merge form — extend the existing `logfire.configure(...)` call

Python:

```python
# Add to the existing logfire.configure(...) call in the user's
# code — do not introduce a second call.
from kubit_otel import KubitSpanProcessor

logfire.configure(
    # …existing kwargs stay in place…,
    additional_span_processors=[
        KubitSpanProcessor(
            api_key=os.environ["KUBIT_EXPORT_API_KEY"],
            token_endpoint=os.environ["KUBIT_EXPORT_ENDPOINT"],
        ),
    ],
)
```

TypeScript: the `@pydantic/logfire-api` surface is smaller and does
not expose an equivalent kwarg; merge by calling `attach()` after the
Logfire initialization call. `attach()` unwraps the proxy provider and
raises if Logfire has not yet initialized, so misordering fails loudly
rather than silently dropping spans:

```typescript
import { attach } from "@kubit-ai/otel";

// Must run after Logfire initialization.
attach({ apiKey: process.env.KUBIT_EXPORT_API_KEY! });
```

### Standalone form — `kubit_instrumentation.py` / `kubit-instrumentation.ts`

Used when §3a finds no `logfire.configure(...)` site to extend.
Attaches on whatever provider Logfire registered globally; must be
imported AFTER Logfire initialization. `attach()` raises if no real
`TracerProvider` is registered yet — so misordering fails loudly
rather than silently creating a second provider that would clobber
Logfire's registration.

Python:

```python
# Generated by /kubit-integrate for logfire on <YYYY-MM-DD>.
# Must be imported AFTER logfire.configure().
# Requires KUBIT_EXPORT_API_KEY and KUBIT_EXPORT_ENDPOINT env vars.
import os
from kubit_otel import attach

attach(
    api_key=os.environ["KUBIT_EXPORT_API_KEY"],
    token_endpoint=os.environ["KUBIT_EXPORT_ENDPOINT"],
)
```

TypeScript:

```typescript
// Generated by /kubit-integrate for logfire on <YYYY-MM-DD>.
// Must be imported AFTER Logfire initialization.
// Requires KUBIT_EXPORT_API_KEY env var.
import { attach } from "@kubit-ai/otel";

attach({ apiKey: process.env.KUBIT_EXPORT_API_KEY! });
```

## 3a. Integration-site signals

Grep targets for an existing wiring site to merge Kubit into. If
exactly one file in the repo matches, merge §3 wiring into that file;
otherwise fall back to the standalone bootstrap file per SKILL.md
step 9.

Python:

- A call to `logfire.configure(` — pass `KubitSpanProcessor(...)` into
  its `additional_span_processors=[...]` kwarg (vendor-documented,
  single call-site edit, no second file).

TypeScript:

- A call to `import logfire from "@pydantic/logfire-api"` plus an
  initialization call. Call `attach({ apiKey: ... })` from
  `@kubit-ai/otel` immediately after the Logfire init in the same
  module.

If multiple files match, ask the user which one the agent's traces
flow through.

## 4. Wire-in instruction

Python: `{{KUBIT_IMPORT_STATEMENT}}` **after** `logfire.configure(...)`
in the same module. Example:

```python
import logfire
logfire.configure()
{{KUBIT_IMPORT_STATEMENT}}  # noqa: E402 — must follow logfire.configure()
```

TypeScript: `import './kubit-instrumentation';` after the Logfire
initialization call in your entrypoint.

Required deps:

- Python: `pip install kubit-otel`
- TypeScript: `npm install @kubit-ai/otel`

## 5. Verification snippet

Python:

```bash
KUBIT_EXPORT_API_KEY=<your-key> KUBIT_EXPORT_ENDPOINT=<your-endpoint> python -c "
{{KUBIT_IMPORT_STATEMENT}}
from opentelemetry import trace
trace.get_tracer('kubit-verify').start_span('hello-kubit').end()
import time; time.sleep(2)
"
```

TypeScript:

```bash
KUBIT_EXPORT_API_KEY=<your-key> node -r ts-node/register -e "
require('./kubit-instrumentation');
const { trace } = require('@opentelemetry/api');
trace.getTracer('kubit-verify').startSpan('hello-kubit').end();
setTimeout(() => process.exit(0), 2000);
"
```
