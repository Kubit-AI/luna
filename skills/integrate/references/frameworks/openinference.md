## 2. Minimum-change tier

`bootstrap-file`

Phoenix owns the `TracerProvider` — `phoenix.otel.register(...)`
constructs it and registers it globally via
`trace.set_tracer_provider(...)`. Python's OTel API enforces a
set-once latch on the global provider, so if kubit-otel registers
first, Phoenix's registration silently becomes a no-op and Phoenix
spans are lost. The standalone snippet in §3 therefore uses `attach()`
and must run **after** `phoenix.otel.register(...)` — it raises
loudly if invoked too early rather than silently dropping spans.

## 3. Bootstrap snippet

Reference Kubit wiring — the *minimum* code that must end up in the
program (import + processor/`configure` call, plus any assertions
below). When merging into an existing wiring site (see §3a), adapt
placement and syntactic style to the surrounding file. The standalone
snippet is used verbatim only when no site is found.

### Python — `kubit_instrumentation.py`

```python
# Generated by /kubit-integrate for openinference on <YYYY-MM-DD>.
# Must be imported AFTER phoenix.otel.register().
# Requires KUBIT_EXPORT_API_KEY and KUBIT_EXPORT_ENDPOINT env vars.
import os
from kubit_otel import attach

attach(
    api_key=os.environ["KUBIT_EXPORT_API_KEY"],
    token_endpoint=os.environ["KUBIT_EXPORT_ENDPOINT"],
)
```

### TypeScript — `kubit-instrumentation.ts`

```typescript
// Generated by /kubit-integrate for openinference on <YYYY-MM-DD>.
// Must be imported AFTER the Phoenix register() call.
// Requires KUBIT_EXPORT_API_KEY env var.
import { attach } from "@kubit-ai/otel";

attach({ apiKey: process.env.KUBIT_EXPORT_API_KEY! });
```

## 3a. Integration-site signals

Grep targets for an existing wiring site to merge Kubit into. If
exactly one file in the repo matches, merge §3 wiring into that file;
otherwise fall back to the standalone bootstrap file per SKILL.md
step 9.

Python:

- A call to `phoenix.otel.register(` — the returned provider is the
  target for `provider.add_span_processor(KubitSpanProcessor(...))`.
  If the call is shadowed in a named variable
  (e.g. `provider = register(...)`), reuse that variable. When no
  named variable is in scope, call `attach(...)` from `kubit_otel`
  on the next line instead — it resolves the provider via the global
  API and raises if `register()` has not run yet.
- A module that already imports `phoenix.otel` and constructs its own
  `TracerProvider` (rare) — merge onto that provider.

TypeScript:

- A call to `register({` imported from `@arizeai/phoenix-otel`.
- A module that constructs its own provider via
  `new NodeTracerProvider(` / `new BasicTracerProvider(` while
  importing from `@arizeai/phoenix-otel` or
  `@arizeai/openinference-*` — merge onto that provider.

If multiple files match, ask the user which one the agent's traces
flow through.
